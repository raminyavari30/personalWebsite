Technologies

Javascript

I am extremely comfortable in Vanilla Javascript. I prefer coding in Vanilla Javascript more than any other framework or library. Having taught Javascript at Hack Reactor / Makersquare, I can be considered exceedingly proficient

HTML

I've worked with HTML as long as I've worked with Vanilla Javascript. I find the layout of HTML to be extremely important, and try to really design my web applications and websites ahead of time. I also try to follow the best practices for HTML layouts by implementing semantic HTML.

CSS

CSS is something I really enjoy doing, and I'm always trying to expand my knowledge in this department. I've worked with CSS frameworks such as Bootstrap, but I personally don't like using them, as CSS frameworks don't challenge me enough to learn the fundamentals of CSS.

Angular

Angular was the first framework I used and became extremely comfortable with. The overhead to get set up with Angular is pretty light, and Angular does a great job applyng the MVC principles of web development.

React

React is doing exciting things to the web development world, with Facebook pushing hard for its adoption. I really like working with this component based framework and think that it's going to become the most popular web framework sooner than later. I've worked on several projects with React, and I've also worked with Redux as well.

Node

Node is the first backend framework I learned, and I absolutely love it. There are so many cool things you can do it Node that sometimes for fun I just sit and read the Node documentation to try to learn more. I have primarily used Node to develop backends and servers.

Portfolio

Caffiend is a web application that allows users to find their new favorite menu item at a given cafe. Users can search for cafes, view menus sorted by item rating, and add their own cafes and menu items.

Caffiend is built in Angular on the front end, with a Node and Express backend that communicates with an SQLite3 database.

CityDash is a simple multitool application that allows users to select specific city related informaton to display on their own customizable personal dashboard.

CityDash is built in React on the front end, with Node and Express on the backend. Firebase stores all the user's data of preferred layouts and other user information.

Itinderary is a web application that generates activities to do based on how the user is feeling. Itinderary cleverly uses the Yelp API combined with its own searching algorithm to return accurate results based on the mood of the user.

Itinderary is built in React, Redux, and React Router on the front end, with Node and Express on the back end.
